# 股票战术执行手册 v1.0

> 最后更新：2026-05-02
> 适用范围：沪深300 + 中证500 成分股

---

## 一、选股池与过滤

| 项目 | 规则 |
|------|------|
| 基础池 | 沪深300 + 中证500 成分股 |
| 排除条件 | ST、\*ST |
| 重大利空过滤 | 近3个月有重大利空（暴雷/立案调查/重大诉讼）→ **由 AI 辅助判断**，标记为人工复核 |
| 选股时机 | 每日收盘后用选股器筛选，次日观察 |
| 关注条件 | 日线 **DEA 最近刚上穿 0 轴**（不要求持续天数，但需"刚上穿"而非长期在0轴上方）+ 收盘价在 60 日线上方 → 等待阴线建仓 |

> **DEA 上穿 0 轴说明**：选股器只需筛选 DEA 刚从负转正的标的。不需要考虑 DEA 下穿 0 轴的情况——如果已持仓，DEA 下穿意味着股价已下跌，应由止损规则（跌破60日线/下穿MA25）处理，而非额外设置 DEA 下穿信号。

## 二、买入规则

### 2.1 入场条件（全部满足）

1. ✅ 日线 DEA 最近刚上穿 0 轴
2. ✅ 当前收盘价在 60 日均线上方
3. ✅ 当日为**真阴线**（同时满足以下两个条件）：
   - 收盘价 < 前一日收盘价
   - 涨跌幅 ≤ **-0.1%**（排除微跌十字星和假阴线）

> **假阴线排除**：高开后收盘价低于开盘价但高于前日收盘价，不算阴线。只有收盘价低于前日收盘价且跌幅超过0.1%的才算真阴线。
>
> **首次建仓与后续加仓的区别**：首次建仓只需满足上述3个条件。后续加仓同样遵循"真阴线买入"规则。

### 2.2 首次建仓

| 距60日线距离 | 买入比例（占计划仓位） |
|-------------|----------------------|
| >1% | **1/3** |
| ≤1%（含踩线） | **全仓**（不再加仓） |

### 2.3 加仓纪律

| 项目 | 规则 |
|------|------|
| 最大买入次数 | **3 次**（含首次建仓） |
| 加仓目标 | 总成本控制在 60 日线附近 |
| 加仓触发 | 每次加仓必须为**真阴线**（同2.1入场条件） |
| 单票上限 | 计划仓位 = 总资金的 **10%** |

**加仓比例（越跌买越多）**：

| 买入批次 | 比例（占计划仓位） | 说明 |
|----------|-------------------|------|
| 第1次（首次建仓） | **1/3** | DEA上穿后出现真阴线 |
| 第2次（加仓） | **1/3** | 价格继续下跌，出现真阴线 |
| 第3次（加仓） | **1/3** | 价格继续下跌，出现真阴线，成本压向MA60 |

> 3次合计 = 1/3 + 1/3 + 1/3 = **100%** 计划仓位。
>
> 若首次建仓已按≤1%规则买入全仓，则不再加仓。

### 2.4 同时持仓上限

| 项目 | 规则 |
|------|------|
| 最大持股数 | **8 只** |
| 总仓位上限 | 80%（8×10%），保留 20% 现金 |

## 三、卖出规则

### 3.1 分批止盈

| 触发条件 | 操作 |
|----------|------|
| **计划总仓位**累计盈利 ≥ 5% | 卖出计划总仓位的 **1/3** |
| **计划总仓位**累计盈利 ≥ 15% | 再卖出计划总仓位的 **1/3** |
| 剩余 1/3 仓位 | 持有，等待清仓信号（MA25下穿或MA60跌破） |

> 📌 **盈利计算基准**：始终按该股票全部已投入资金的**原始总成本**计算（加权平均成本），不因卖出而调整基准。
>
> 📌 **卖出对象**：每次卖出的是**计划总仓位的 1/3**，而非当前持仓的 1/3。
>
> 📌 **第一次止盈后价格回落**：卖出1/3后若价格回落至5%以下，剩余仓位继续持有，监控规则不变——仍以 **MA60 为止损线、MA25 为清仓线**。不会重新触发第一次止盈。

### 3.2 清仓信号

| 触发条件 | 操作 |
|----------|------|
| 日线**收盘价下穿 25 日均线（MA25）**，且**次日收盘仍未收回MA25上方** | **全部清仓**（含剩余 1/3 仓位） |

> 收盘跌破MA25当天不操作，次日收盘观察——若收回MA25上方则继续持有，若仍未收回则执行清仓。

### 3.3 止损规则

| 触发条件 | 操作 |
|----------|------|
| 日线**收盘价跌破 60 日均线**，且**次日收盘仍未收回MA60上方** | **全部清仓**（无论盈亏） |

> 收盘跌破MA60当天不操作，次日收盘观察——若收回MA60上方则继续持有，若仍未收回则执行清仓。
>
> ⚠️ 止损优先级高于止盈——如果同时触发止盈和止损，执行止损

### 3.4 时间止损

| 持仓时长 | 触发条件 | 操作 |
|----------|----------|------|
| 30 个交易日 | 累计盈利未超过 3% | **全部清仓** |

> ⚠️ 30日清仓后，若60日均线未跌破，理论上可重新入场（需重新满足入场条件），但建议至少观察5个交易日再考虑。

### 3.5 涨停/跌停处理

| 场景 | 操作 |
|------|------|
| **准备买入当天开盘涨停** | 不买入，放弃本次信号 |
| **触发止损但当日跌停卖不出** | 次日开盘继续执行卖出，不等待反弹 |
| **一字跌停** | 不买入（即使满足其他入场条件），等跌停打开且重新满足条件后再考虑 |
| **买入价格** | 必须在 60 日均线上方成交，盘中跌破60日线但收盘收回的不算止损 |

## 四、板块辅助判断（可选增强）

> 当前版本不作为硬性过滤条件，仅作参考

- 买入前观察该股所属**板块指数**的趋势
- 若板块指数处于明确下跌趋势，谨慎建仓
- 若板块指数同步走强，可提高优先级

## 五、执行流程图

```
每日收盘后
    │
    ▼
选股器筛选（沪深300+中证500，AI辅助排除近3月重大利空）
    │
    ▼
是否有 DEA 刚上穿 0 轴 + 收盘在60日线上方的标的？
    │
   YES ────── NO ──→ 不操作
    │
    ▼
当日是否为真阴线？（收盘<前日收盘，跌幅≥0.1%，非假阴线）
    │
   YES ────── NO ──→ 继续等待阴线
    │
    ▼
是否开盘涨停？一字跌停？
    │
   NO（正常交易）── YES ──→ 不买入
    │
    ▼
距60日线距离判断 → ≤1%则全仓，否则买1/3
    │
    ▼
检查：是否已有8只持仓？
    │
   YES ──→ 不开新仓
   NO ───→ 执行买入
    │
    ▼
持仓监控（每日收盘后检查）
    │
    ├─→ 收盘跌破60日线 → 次日观察，仍未收回 → 清仓 🔴（跌停则再延一天卖）
    ├─→ 收盘下穿MA25 → 次日观察，仍未收回 → 清仓 🔴
    ├─→ 30日盈利<3% → 清仓 ⚠️
    ├─→ 60日未止盈 → 清仓 ⚠️
    ├─→ 总成本盈利≥5% → 卖1/3 ✅
    ├─→ 总成本盈利≥15% → 再卖1/3 ✅
    └─→ 真阴线 + 未满3次 → 按比例加仓（越跌越多）
```

## 六、关键参数速查表

| 参数 | 数值 |
|------|------|
| DEA 刚上穿 0 轴 | 入场信号（刚上穿，非长期在0轴上方） |
| 60日均线 | 买入参考线 + 止损线（需次日确认） |
| 25日均线 | 清仓信号线（需次日确认） |
| 单票计划仓位 | 总资金 10% |
| 最大持股数 | 8 只 |
| 首次建仓 | 1/3（距MA60≤1%则全仓） |
| 最大买入次数 | 3 次（1/3 + 1/3 + 1/3） |
| 止盈1 | 盈利 5% → 卖 1/3 |
| 止盈2 | 盈利 15% → 再卖 1/3 |
| 时间止损1 | 30日盈利<3% → 清仓 |
| 时间止损2 | 60日无止盈 → 清仓 |
| 真阴线定义 | 收盘价 < 前日收盘价，且跌幅 ≥ 0.1% |
| 止损确认 | 收盘跌破MA60 → 次日收盘仍未收回 → 清仓 |
| 清仓确认 | 收盘跌破MA25 → 次日收盘仍未收回 → 清仓 |
| 涨停处理 | 开盘涨停不买 |
| 跌停处理 | 触发止损但跌停卖不出 → 次日卖；一字跌停不买 |
| 重大利空 | AI 辅助判断，标记人工复核 |

## 七、风险提示

1. **本战术属于趋势跟踪 + 均线回归策略的混合体**，核心假设是"DEA上穿0轴后趋势延续"
2. **DEA上穿0轴存在滞后性**，可能错过最佳入场点，买入时已处于趋势中段
3. **"越跌越买"的逆势加仓逻辑**在极端行情下风险较大（如2022年4月、2024年1月的系统性下跌）
4. **60日线止损是最后防线**，如果出现跳空低开直接破60日线，实际止损点可能远差于预期。加上次日确认机制后，极端连续下跌中可能多承受一天损失
5. **涨停/跌停限制**：极端行情下可能无法按计划买入或止损，需接受实际执行偏差
6. **重大利空过滤依赖 AI 判断**，可能存在误判，建议对 AI 标记的标的进行人工复核
7. **建议先用模拟盘或小仓位验证至少3个月**，记录每笔交易的执行情况，再逐步加仓
