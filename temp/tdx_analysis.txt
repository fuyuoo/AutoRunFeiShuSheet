## 2. PriCS.dat ANALYSIS (802,604 bytes)

- CONSTANT file size - stores compiled formula parameters, NOT source code
- Header: 05 00 00 00 7c 01 df 00 6b 00 0f 00 23 00 00 00
- Contains M1-M8 named parameter blocks with IEEE 754 float values
- Each M-block has 20+ float parameters (M1 at 0x0030, M2 at 0x00B0, etc.)
- Readable parameter names: M1-M8, SHORT, LONG, MID, AVGLENGTH, ATRLENGTH, NMIN1, NMIN2, STEP, MAXP
- GBK Chinese characters partially visible but garbled in binary
- Conclusion: Parameter/settings storage, NOT formula source

## 5. .cos FILES (94 bytes each)

- b1_check.cos: Header 00 02 00 00 01 00 00 00, name "B1_CHECK", 3 float params
- b2_check.cos: Same header, name "B2_CHECK", 3 float params
- 宝妈战法.cos: Same header, name field all zeros (uninitialized or stored differently)

## 7. .tn6 FORMAT ANALYSIS (from temp/ files)

### File: 123.tn6 (44,752 bytes)
- 64-byte record structure (699 records + 16 byte remainder)
- 8-byte null placeholder pattern: 00f736f388946906
- 8-byte marker pattern: 234b04de8ac7eb38 (marks record boundaries)

### Record Type Classification:
- Records starting with marker (234b04de8ac7eb38): formula entry points
- Records starting with null pattern (00f736f388946906): unused/empty slots
- Records starting with zeros: all-zero blocks
- Mixed records: actual compiled formula data

### XOR Decoding Attempts:
- XOR with null pattern key did NOT reveal readable source code
- Encoding is more complex than simple XOR
- The null pattern (00f736f388946906) itself XORs to f7c0060547064132 - not meaningful

### Key Insight:
The .tn6 file contains compiled/formula-processed data, NOT source code text.
The formula source code is stored in the TDX application's internal memory/files
and compiled on-the-fly by TEvalExp.dll + TCalc64.dll when formulas are loaded.

## 8. CONCLUSIONS

1. **No formula source code exists as readable text** anywhere in the TDX installation
2. **PriGS.dat contains compiled bytecode** - the closest thing to "formula code" but in proprietary binary format
3. **The formula compiler lives in TEvalExp.dll + TCalc64.dll**, invoked by TdxW.exe
4. **.tn6 files are export artifacts** created through the TDX UI, not stored in the installation
5. **To decompile formulas**, one would need to:
   - Reverse-engineer the TCalc64.dll bytecode format
   - Or capture formulas in source form from TDX's editor before compilation
   - Or use TDX's built-in "formula sharing" features to export readable versions
6. **The gs_bak/ backups** are the best way to track formula changes over time
7. **.cos files** are simple parameter snapshots (94 bytes), useful for tracking parameter changes

## 9. RECOMMENDATIONS FOR FURTHER WORK

- Use TDX's built-in formula export/sharing features to get readable source
- Monitor gs_bak/ for daily formula backups
- The .tn6 format may be decipherable by comparing source + compiled versions
- Consider using IDA Pro/Ghidra on TCalc64.dll to understand the bytecode format
- The "downloadformula"/"uploadformula" strings in TdxW.exe suggest network-based formula transfer
